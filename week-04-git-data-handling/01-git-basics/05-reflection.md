# 05. Reflection: Git Basics

## Recall Questions

1. What is the difference between staging and committing?
2. Why is a clear commit message useful?
3. What do `git push` and `git pull` do?
4. What does imperative commit style mean in your own words?

## Micro Reflection

- One Git command that now feels clear.
- One Git mistake I made and what caused it.
- How I fixed that mistake.
- One commit message habit I will keep.

## Applied Practice (Optional)

Write three lowercase imperative commit messages for a small data project: one for setup, one for file handling, and one
for bug fix.
